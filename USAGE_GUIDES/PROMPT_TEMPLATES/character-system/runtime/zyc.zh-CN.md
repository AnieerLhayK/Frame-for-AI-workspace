# zyc Prompt 模板

在 Codex 或 OpenCode 中使用——前提是 `zyc` 已在该平台的技能列表中可见。该技能在两个平台上均保持纯文本模式。

## 自然讨论

当你希望 ZYC 以数字人格风格的声音回应，而非像写作工具菜单时，优先使用此模板。

```text
使用 zyc。

请对以下想法进行自然回应。

不要让我选择 rewrite / continuation / critique / discussion。
不要使用标题、项目符号列表、表格或评分。
不要用讨好式 AI 口吻总结我的观点。
你可以酌情同意、犹豫、质疑或反对。
保持风格启发，而非身份模仿。

输入：
<text>
```

## 用 ZYC 风格改写

```text
使用 zyc。

任务：
改写以下文本，保留其含义。

输入：
<text>

约束：
- 保持输出为风格启发，而非身份模仿。
- 不要添加私人事实。
- 不要过度解释风格。
- 保留用户要求的语言和任务。

输出格式：
除非我要求附注，否则仅返回改写后的文本。
```

## 续写片段

```text
使用 zyc。

任务：
以兼容的语气和节奏续写此片段。

片段：
<text>

约束：
- 延续已有的情感和节奏方向。
- 不要总结该片段。
- 不要解释模仿。
- 不要添加私人传记性声称。
```

## 如果输出感觉不对

```text
使用 style-doctor。

目标技能：
zyc

观察到的问题：
<什么感觉不对>

失败输出片段：
<片段>

预期方向：
<本应发生什么>

任务：
诊断漂移。不要直接修补 zyc。
```
