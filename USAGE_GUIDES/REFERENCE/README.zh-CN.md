# 参考指南

在快速启动或提示词注册表确定你要做什么之后，使用本索引。仅阅读当前任务所需的章节。

本层说明职责、边界和选择标准。它不是可直接复制使用的提示词层。当你需要可直接粘贴到AI会话中的文本时，请使用 `../PROMPT_TEMPLATES/` 或通过 `../prompt_registry.yaml` 解析提示词 ID。

## 运行时

当使用面向用户的技能或决定观察到的输出是否应交由诊断或维护时，打开 `runtime/`。

## 工程

当生成、诊断、维护或演进一个技能时，打开 `engineering/`。子目录标识工程职责：

- `generation/`：根据授权输入创建或更新生成的技能制品。
- `diagnosis/`：检查输出并生成有边界的证据或交接包。
- `maintenance/`：修补现有技能，但不接管生成器的所有权。

当多个工程角色都可能拥有该变更的所有权时，使用 `engineering/production-maintenance-relationships.md`。

## 工作流

对于跨越运行时和工程角色（包括生成、漂移诊断、修补、验证和生成器泛化）的端到端流程，打开 `workflows/`。

## 平台

仅针对特定平台的加载和安全性细节，打开 `platforms/`。平台暴露决定技能的可见位置；它不决定技能的来源所有权或修改权限。

关于规范的源路径和当前暴露情况，请参考 `workspace_manifest.yaml`，而不是将路径信息复制到本指南中。

当本指南和提示词模板涵盖同一工作流时，请保持本指南简短且具解释性。将详细的分步护栏放在匹配的模板中。
