# Case Studies

Use this directory for real cases after they happen.

Good case studies include:

- Agent registration and workspace governance changes;
- source, projection, Session, cache, and output boundary incidents;
- successful skill evolution;
- failed governance attempts;
- runtime drift repairs;
- portability problems;
- validator findings;
- runtime diagnosis examples.

Do not invent case studies to make the knowledge base look complete. Empty is better than fictional history.

Each case should state its date, evidence level, problem, design, verification,
reusable lessons, and known limits. External influences must follow
`../knowledge_provenance.md`.
