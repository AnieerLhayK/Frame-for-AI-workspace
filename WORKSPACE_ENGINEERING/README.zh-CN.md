# 工作空间工程知识库

`WORKSPACE_ENGINEERING/` 是用于设计、运行、演进和共享 AI 工作空间的可复用工程知识层。

它涵盖的内容比技能（Skill）构建更为广泛：

- 工作空间与仓库架构；
- Agent 身份、注册、权限与生命周期；
- 源、投影、数据、缓存、会话与输出边界；
- 任务路由、上下文预算与开发者接口；
- 治理、验证、报告、可移植性及发布实践；
- 技能工程（Skill Engineering）作为其中一个专业子领域。

该层是一本基于经验积累的参考手册。它不代表当前工作空间状态，不是可执行策略，不是原始链接集合，也不声称每种模式普遍正确。

## 与其他层次的关系

- `PROJECT_CONTEXT/`：当前工作空间状态、活跃记忆、任务与交接信息。
- `shared/`：本工作空间的可执行契约。
- `workspace_manifest.yaml`：机器可读的当前工作空间事实。
- `reports/`：生成或编写的工作快照。
- `WORKSPACE_ENGINEERING/`：可复用的方法论与有证据支撑的经验教训。

当这些层对当前行为描述不一致时，知识库不覆盖清单（Manifest）、策略、源代码或当前 Git 状态。

## 手册结构

### 工作空间工程

- `philosophy.md`：通用工程姿态。
- `architecture_patterns.md`：可复用的系统结构。
- `workspace_patterns.md`：仓库与工作空间组织。
- `governance_patterns.md`：权限、验证与连续性。
- `portability_patterns.md`：迁移与机器边界相关的经验教训。
- `anti_patterns.md`：反复出现的失败模式。
- `knowledge_provenance.md`：知识引入与归属规则。

### 外部知识

`external_knowledge/` 包含关于引入外部知识检索（RAG、知识库、笔记系统）的有限评估，不涉及具体的实施或部署。

- `external_rag_planning.md`：分阶段评估与工具对比。

### 技能工程

`skill_engineering/` 包含可复用的技能专用方法：

- 技能设计；
- 提示词工程；
- 运行时循环；
- 漂移与风格对齐；
- 经过验证的演进模式。

### 证据

- `case_studies/`：真实事件与已完成变更。
- `experiments/`：有限假设与结果；当前处于休眠状态，待开展真实实验。
- `templates/`：可复用的编写结构。
- `reports/`：关于该知识层的快照。

## 何时添加知识

仅在已完成的任务至少产出一项以下内容时添加条目：

1. 可复用的模式或决策框架。
2. 反复出现的反模式。
3. 真实的事件或迁移案例。
4. 经过验证或被推翻的实验。
5. 对长期风险或架构产生实质性影响的变化。

常规任务状态保留在 `PROJECT_CONTEXT/task_ledger.md` 中。

## 完成时的回写检查

在工作空间维护任务结束时，Agent 应做出简短的知识回写决策：

- **回写**：当任务产出了可复用的方法、反模式、事件、迁移经验、实验结果或架构权衡时。
- **记录无需回写**：当任务仅改变了局部状态、应用了现有规则、刷新了报告或修复了狭窄的缺陷而未产出可复用的经验教训时。

选择最小的持久化存放位置。将操作连续性信息放入 `PROJECT_CONTEXT/task_ledger.md`，将可复用的方法论放入本目录，仅当策略契约实际发生变化时才将可执行规则放入 `shared/`。

## 证据等级

每项新经验教训应明确其证据等级：

- **已观察到**：在真实环境中见过但尚未复现。
- **局部验证**：在本工作空间或其他受控项目中测试过。
- **重复验证**：在多个任务或环境中得到确认。
- **外部参考**：改编自被引用的来源，尚未在本环境中完全验证。
- **已退休**：为历史保留但不再推荐。

## 共享原则

编写时应让另一位 AI 工程师无需访问本机器的私有数据即可理解上下文、权衡、失败模式和验证步骤。从可共享的指南中移除密钥、私有语料材料、本地身份详情和易变的绝对路径。

外部思想仅能通过 `knowledge_provenance.md` 引入。将其总结并转化为经过本地审核的工程知识；不得复制大量第三方材料或抹去其原始出处。
