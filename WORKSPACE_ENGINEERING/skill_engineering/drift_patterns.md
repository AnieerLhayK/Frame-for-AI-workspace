# Drift Patterns

Drift is any movement away from the intended contract, behavior, structure, or source-of-truth relationship.

## Taxonomy Drift

Occurs when categories stop matching real failures. Fix by updating taxonomy only after repeated evidence.

## Protocol Drift

Occurs when skills reference old or missing shared protocols. Fix with protocol validator and `SHARED_PROTOCOLS.md` review.

## Report Drift

Occurs when reports are mistaken for current truth. Fix with snapshot headers and regeneration when needed.

## Path Drift

Occurs when files or docs point to old roots or projection paths. Fix by using manifest data and bounded bootstrap.

## Evolution Drift

Occurs when skill patches accumulate without a coherent history. Fix through runtime loop records and ledgers.

## Platform Drift

Occurs when Codex and OpenCode projection surfaces diverge from source, or when exposure is mistaken for ownership or authority. Fix through link checks, manifest exposure validation, and source-first edits.

## Schema Drift

Occurs when templates, packet fields, and schema files diverge. Fix by aligning schemas with templates and keeping validation lightweight at first.

## Prompt Drift

Occurs when prompts lose boundaries, source-of-truth reminders, or final-report requirements. Fix by preserving prompt contracts for high-risk governance work.
